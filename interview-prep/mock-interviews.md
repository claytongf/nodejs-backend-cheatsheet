# Mock Interviews

Use these formats to practice under time pressure. Record yourself if possible.

## 30-Minute Junior Backend Mock

1. **5 min:** Explain the Task Manager API at a high level.
2. **10 min:** Answer HTTP, Express, and validation questions.
3. **10 min:** Walk through auth and one protected endpoint.
4. **5 min:** Explain one test and one thing you would improve.

Passing signal: clear fundamentals, correct `401` vs `403`, and ability to point to real
files.

## 45-Minute Mid-Level Backend Mock

1. **5 min:** Project walkthrough.
2. **10 min:** Architecture and layering discussion.
3. **10 min:** Database and Prisma discussion.
4. **10 min:** Auth, authorization, and IDOR discussion.
5. **5 min:** Testing strategy.
6. **5 min:** Production trade-offs.

Passing signal: explains trade-offs and failure modes, not only definitions.

## 60-Minute Practical Mock

1. **10 min:** Project walkthrough.
2. **20 min:** Live coding prompt from [live-coding.md](live-coding.md).
3. **10 min:** Debugging prompt from [debugging-challenges.md](debugging-challenges.md).
4. **10 min:** System design extension from [system-design.md](system-design.md).
5. **10 min:** Behavioral questions from [behavioral.md](behavioral.md).

Passing signal: makes a reasonable plan, keeps changes scoped, tests the result, and
communicates trade-offs.

## Debrief Checklist

- Did I answer the actual question?
- Did I use concrete project examples?
- Did I mention trade-offs where appropriate?
- Did I avoid pretending to know what I do not know?
- Did I explain how I would verify the answer?
